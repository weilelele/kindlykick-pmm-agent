// 设计者说

为什么造了这个东西

大 P 人 大 FP 人 大 ENFP

我非常重视如何推进项目，同时极度看重情绪价值——保证大家情绪高涨，对我来说和进度一样重要。
但让我自己人工管理进度？非常难。无数种科学管理方法，在大厂里学过、用过很多， 真正要管一个团队的那天我才明白：我需要一个绝对铁面无私的助手。

"如果不能激发人与人之间的连接，再高效的工具也只是一台冷冰冰的机器。"

// V1 · 踢屁股专家

能高效整理工作内容，能 push 大家、催进度。
但我发现，他与人之间的连接变弱了——没法激发大家互动的欲望，
事件上报不及时，效率也随之变差。

// V2 · 彩虹屁踢屁股专家

激发团队潜力的关键，是能够「看见」他——
让他的行为和感受被看见、被尊重。
于是我把彩虹屁这个特质加了上去，才有了现在的这个产品。

01

既能夸夸大家、真心看见大家的付出，又不断踢大家屁股，时刻跟进项目日程。

02

完美融入日常工作流——把所有文档和想法一股脑丢给他，任务自动整理、通知到人。

03

团队专属的PMM（项目营销经理）：你只需要提供素材，他负责让所有人动起来。

// 设计背景

为什么需要一个
彩虹屁踢屁股专家？

📋

任务散落在会议记录里

每次开完会，有人记录在文档里，有人记在脑子里。一周后没人记得谁负责什么，截止日期也成了摆设。

👻

进展更新靠"催"

项目管理者每天手动追进度，发消息问"这个做完了吗"，得到的回复往往是沉默或一句"在做了"。

😶

努力没有被看见

团队成员默默做事，没有反馈，没有认可。时间久了，积极性自然下降——不是不努力，是感觉没人在意。

🛠

工具太重、配置太多

现有 PM 工具需要大量配置和上手培训。小团队需要的是开箱即用、用自然语言就能操作的助手。

// 核心特质

两个特质，缺一不可

踢屁股

催进度、盯截止日、对拖延零容忍。
每晚 18:00 准时播报，没有进展的直接点名施压。 不说废话，不讲情面，只问结果。

"乐乐，今天约好的 12:00 音乐朋友那边聊了吗？结果呢？"
彩虹屁

真心看见每个人的努力。不是泛泛的"加油"，
而是具体说出他做了什么、为什么值得被看见。
每次汇报、每次播报，都有一句话留给这个人。 让团队感受到：我不只是一台完成任务的机器。

"三天磨一件事还没让它凉——这种耐心很多人做不到，漂亮。"

// 获取渠道

五分钟部署上线

01

建好基础设施

在 Supabase 新建项目并运行 schema.sql 建表。申请 Anthropic API Key，在飞书开放平台创建企业自建应用并开启机器人。

02

一键部署 Railway

Fork GitHub 仓库后连接 Railway，填入环境变量（Feishu / Supabase / Anthropic），推送即部署。把 Railway 域名填回飞书 Webhook。

03

把 Bot 拉进群

Bot 自动发欢迎消息，大家轮流 @ 它做自我介绍。之后把会议记录发给它，任务自动提取，当天就能开始追进度。

开源 · 免费 · 自部署

所有代码开放在 GitHub，按实际 API 用量付费，小团队每月约 $2。
